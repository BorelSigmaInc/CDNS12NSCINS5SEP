ML Engineer

Compensation: Paid · Timeline: 3–5 months · Git Enabled: Yes

Train, fine-tune and evaluate code-generation models; design repeatable Pass@k pipelines.

Responsibilities

  • Implement training/eval with PyTorch; maintain prompts and unit tests.
  • Track model checkpoints, metrics and regressions.
  • Collaborate with Data, DevOps and QA for reproducible runs.

Requirements

  • Python, PyTorch/TensorFlow; familiarity with evaluation datasets.
  • Experience with OpenAI API or similar LLM tooling.

Preferred Qualifications

  • Hands-on with mixed-precision (AMP) and Distributed Data Parallel.
  • Experience running multi-sample evaluations and aggregating Pass@k.
  • Exposure to tokenization internals (e.g., HF Tokenizers).
  • Basic Kubernetes literacy for reproducible execution.

Methodology & Practices

We favor transparent, reproducible experiments. Typical runs include:

  • Seeded training/eval with pinned dependencies and environment hashes.
  • Checkpoint versioning with metrics, prompts, and data snapshot IDs.
  • Sandboxed evaluation to mitigate side effects, plus flaky-test tracking.
  • Weekly regression reviews and ablations to diagnose failure modes.

Suggested Reading

  • PyTorch AMP and DDP tutorial
  • Attention Is All You Need (Transformers)
  • Pass@k Metric and evaluation notes
  • OpenAI API usage guidelines
